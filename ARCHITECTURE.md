## Data Flow

### Guest Analysis
```
User pastes text в†’ AnalysisForm (client)
  в†’ localStorage counter check (< 5: proceed, = 5: show GateModal)
  в†’ POST /api/analyse
      в†’ validate length (50вЂ“10,000 chars)
      в†’ ratelimit check (IP)
      в†’ fleschKincaid(text) в†’ score + label
      в†’ openai.analyseText(text) в†’ {summary, keywords}
      в†’ if session.user: prisma.analysis.create(...)
      в†’ return {summary, keywords, readabilityScore, readabilityLabel}
  в†’ ResultsPanel renders output
```

### Auth Flow
```
Signup в†’ POST /api/auth/signup (custom) or NextAuth register action
  в†’ bcrypt.hash(password)
  в†’ prisma.user.create(...)
  в†’ generateToken() в†’ prisma.verificationToken.create(...)
  в†’ resend.sendVerificationEmail(email, token)
  в†’ redirect /auth/verify-email

Email link click в†’ GET /auth/verify-email?token=...
  в†’ hashToken(token) в†’ prisma.verificationToken.findUnique()
  в†’ check expiry в†’ prisma.user.update({emailVerified: true})
  в†’ prisma.verificationToken.delete()
  в†’ redirect /auth/login
```

## Key Architectural Decisions

1. **Single Next.js project**: API routes colocate with UI; no separate Express server needed at this scale.
2. **Prisma adapter for NextAuth**: Users live in our own DB; no Supabase Auth used вЂ” only Supabase as managed Postgres host.
3. **One OpenAI call per analysis**: System prompt requests both summary and keywords as structured JSON to minimize latency and cost.
4. **Guest analyses not persisted**: Reduces DB write load; history is a logged-in-only feature.
5. **httpOnly cookie session**: NextAuth stores JWT in httpOnly cookie; no localStorage token exposure.
6. **Readability computed server-side without AI**: Deterministic, free, fast вЂ” no reason to use a model for this.
7. **Rate limiting on /api/analyse**: Most abuse-prone endpoint; IP-based limit of 20 req/min.
